1. 需求来源和启发

昨天在公司第二次做导入功能的时候，想着界面也和第一次做的差不多，换换接口就完事的东西，

没想到最后在有前端页面的情况下只处理数据和连接整个逻辑流程就花了一天的工时。

就因为有个需求是： 可以选择excel的sheet和显示所选择的sheet中数据的条数.

第一次写的时候没考虑那么多，就直接用 SheetJS 读。

emmm…

数据小效果还不错，但是我们还做了个大表格测试，此表有3个sheet 每个sheet有10000条数据

一选文件屏幕直接卡死， （界面阻塞了好久，直接到卡到页面无响应！）

学过js的都知道它它它在浏览器上是没有多线程的概念的(⊙﹏⊙)

然后我就在想, 那能不能像Ajax那样实现异步读取勒？

想法有了，上👉Google 此处省略N个字….

最后找到了它👉 Worker

2. 解决方案

Worker:

​ Worker 接口是 Web Workers API 的一部分，指的是一种可由脚本创建的后台任务，任务执行中可以向其创建者收发信息。要创建一个 Worker ，只须调用 Worker(URL) 构造函数，函数参数 URL 为指定的脚本。

Worker 也可以创建新的 Worker，当然，所有 Worker 必须与其创建者同源（注意：Blink暂时不支持嵌套 Worker）。

【上面是官方文档说的】

再看看官方示例：

var myWorker = new Worker('worker.js');
var first = document.querySelector('#number1');
var second = document.querySelector('#number2');

first.onchange = function() {
  myWorker.postMessage([first.value,second.value]);
  console.log('Message posted to worker');
}
// https://developer.mozilla.org/zh-CN/docs/Web/API/Worker/postMessage
// worker.js  [文档并没看到这个，自己加上的😀] （后来发现在里面一级目录）
self.addEventListener('message', ({data})=>{
    const [firstVal, secondVal] = data
    console.log(firstVal, secondVal)
    // do something...
    // 做完了可以告诉主线程， 
    // 参数的话：postMessage(message: any, options?: PostMessageOptions): void;
    self.postMessage('我做完了~')
    // 也就可以这样， ！！！但是不可以传函数，类的话只能传实例（当然也有人做了那种将函数转成字符串再传的）
    self.postMessage({message: '我做完了~', data: [firstVal, secondVal]})
})

        // 这是业务代码
        const {type, result} = data
        switch (type) {
          case 'sheet-length':
            this.sheetLength = result
            break
          case 'excel-info':
            this.sheetNames = result.SheetNames
            this.selectSheet(1) //默认选择第一条
            break
          case 'error':
            this.$errorMsg('操作Excel异常!')
            console.warn('[warn]操作Excel异常! message: ' + result)
            break
        }
        this.reading = false
      })
    },
    beforeDestroy() {
      if (!this.worker) return
      this.worker.postMessage({type: 'destroy'})
      this.worker.terminate()
    },
    methods:{
      startReadFile(file) {
        const payload = {
          file: file.raw,
          rABS: false
        }
        this.reading = true
        this.worker.postMessage({type: 'getExcelInfo', payload})
      },
      getSheetLength(sheetName){
        this.worker.postMessage({type: 'getSheetLength', payload: sheetName})

if (!self.XLSX) {
  // 加载工具
  self.importScripts('../js/xlsx.full.min.js')
}
const runtimeFactory = {
  _finished(type, result) {
    type = type || 'finish'
    self.postMessage({type, result})
  },
  _bin2str(data) {
    let o = '', l = 0, w = 10240
    for (; l < data.byteLength / w; ++l)
      o += String.fromCharCode.apply(null, new Uint8Array(data.slice(l * w, l * w + w)))
    o += String.fromCharCode.apply(null, new Uint8Array(data.slice(l * w)));
    return o
  },
  _readFile(payload, callback) {
    const {file, rABS} = payload;
    const reader = new FileReader()
    reader[rABS ? 'readAsArrayBuffer' : 'readAsBinaryString'](file)
    reader.onload = ev => callback && callback(ev.target.result, ev)
  },
  getExcelInfo(payload) {
    const {rABS} = payload
    if (!self.XLSX)
      throw new Error('Require Import SheetJS! More Information:http://sheetjs.com')
    this._readFile(payload, result => {
      const {SheetNames, Sheets} = rABS
        ? self.XLSX.read(btoa(this._bin2str(result)), {type: 'base64'})
        : self.XLSX.read(result, {type: 'binary'})
      this._finished('excel-info', {SheetNames})
      self.Sheets = Sheets
    })
  },
  getSheetLength(sheetName) {
    let sheetLength = 0
    try {
      //解析单元格范围
      const sheet = self.Sheets[sheetName]
      if(!sheet) return
      const range = self.XLSX.utils.decode_range(sheet['!ref'])
      //获取行数（数据条数不包含表头）
      sheetLength = range.e.r - range.s.r
    } catch (e) {
      console.warn(e.message)
      sheetLength = 0
    } finally {
      this._finished('sheet-length', sheetLength)
    }
  }
}

self.addEventListener('message', ({data}) => {
  try {
    const {type, payload} = data
    if (type === 'destroy') {
      self.Sheets = null
      self.close()
    }
    const registeredTypeList = [
      'getExcelInfo',
      'getSheetLength'
    ]
    if (!registeredTypeList.includes(type))
      throw new Error('Unsupported Message!')
    runtimeFactory[type].call(runtimeFactory, payload)
  } catch (e) {
    self.postMessage({type: 'error', result: e.message})
    console.warn(e)
  }
})

3. 总结

代码勒就这么多，难度不大，

但是我做的时候碰到一个问题：

我之前做的是在getExcelInfo函数下直接把XLSX读到的数据全部返回给主线程，

读表格的时候还挺好，读完了界面直接卡死！！！

原因就是读到的数据太大了，postMessage的时候卡死了

所以像Worker这个东西，还是内部处理好大数据量的所有事情，就返回一个状态标记之类的东西给主线程就好了

​ 那如果需要读一个大表格然后展示到页面上使用Worker怎么做呢，欢迎留下你的思路~
